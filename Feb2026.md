- 개별적으로 어떤 요청이 있을 때마다 실시간으로 통신하는 것이 아닌, 한꺼번에 일괄적으로 대량 건을 처리한다. 정해진 특정 시간에 실행되도록 스케줄 진행\

    `특정 시간에 실행`\
    `일괄적으로 처리`

- Cron
리눅스의 스케줄로 도구로 `crontab` 명령어를 통해 작업 예약

## 기술부채

- 정의: 소프트웨어 개발에서 장기적인 품질보다 빠른 납품을 위해 쉬운(임시) 해결책을 선택했을 때 발생하는 향후 재작업 비용

- 기업은 부채를 통해 더 많은 투자를 집행하고, 시장점유율을 높일 수 있다.

- 발생 이유
  - 개인에 의해 발생: 성장하는 개발자(이전보다 더 좋은 방식에 대해 잘 알게 됨)

  - 팀에 의해 발생: 협업하는 팀. 다른 사람의 코드 구조가 자신의 마음에 안드는 경우, 생산성 저해하는 부채로 생각함.

  - 서비스: 발전하는 서비스. 기본 서비스를 빠르게 오픈하고 향후 추가 기능을 개발해 릴리즈.
  

- 최소화 방법
  - 컨벤션 설정(코딩컨벤션, 코드리뷰)
  - 리팩터링: 결과 변경 없이 코드 구조 재조정해서 실재적인 부채 갚기
  테스트 코드 작성: 테스트 주도 개발(TDD, Test Dri ven Development): 기능 개발 시 테스트 코드부터 작성

## 블록체인

### Web3.0

1. 개념: 
<br>
기존 중앙화된 웹(Web2.0)과는 달리, 블록체인과 분산 원장 기술을 기반으로 해 데이터를 분산 저장하고, 스마트 계약을 통해 자동화된 거래와 상호작용을 가능케 함. 인터넷 사용자가 자신의 데이터와 디지털 자산을 직접 소유하고 통제할 수 있도록 하며, 중개자를 배제한 직접적인 거래와 상호작용을 촉진함. 이는 사용자가 자신의 데이터와 자산에 대한 권리를 강화하고, 데이터 보안과 프라이버시를 보호하는 데 중요한 역할을 함.

2. 핵심 기술:
<br>
블록체인, 스마트 계약, 분산 애플리케이션(DApps), 탈중앙화 자율 조직(DAO), 탈중앙화 금융(DeFi), 분산 저장, 인터오퍼러빌리티, 암호화 기술 등으로 구성됨.

  - 블록체인" 데이터를 분산된 네트워크에 블록 형태로 저장하는 분산 원장 기술. 모든 거래를 투명하게 기록하고, 중앙 기관 없이도 거래의 신뢰성과 무결성을 보장함. 이는 데이터 위변조를 방지하고, 분산된 네트워크 참여자 간의 신뢰를 구축하는 데에 중요한 역할을 함. 블록체인은 금융 거래, 공급망 관리, 디지털 자산 소유권 관리등 다양한 분야에서 활용됨.

## HTTP 통신과 Web Socket

### 1. HTTP
Client 통신이 있을 때만 Server에서 응답을 주는 단방향 통신.
Statefuless protocol - Server가 Client 상태 저장 안함(Connect/Close 반복)

### 2. Socket 통신

- Server Client가 특정 port 열어서 실시간으로 양방향 통신을 함

- Stateful Protocol - Server가 Client 상태를 저장함
(임의로 연결 끊지 않는 한 서로 연결 유지)

- DB를 통해 데이터 주고 받을 수도 있고, 임시로 데이터 서버 프로그램 자체 내에서 생성해서 갖고 있을 수도 있음.

- 서버 자체의 메모리에 유저 데이터 저장함


*Socket과 WebSocket과는 다르니 주의

*TCP: 데이터 전송 시, 상대방에게 정확히 갈 수 있는지 체크(정합성)

*UDP: 보내기만하고 제대로 전달됐는지 체크할 수 없다. ex) 채팅(유실돼도 크게 문제는 없음)
