### 234. C언어의 구조체
- 배열: 자료의 형과 크기가 동일한 변수의 모임
- 구조체: 자료의 종류가 다른 변수의 모임
  - ex. 이름, 직위, 급여 등 필드가 필요한 사원 자료를 하나의 단위로 관리하려면, 이름과 직위는 문자, 급여는 숫자 등 혼용되므로 배열로는 처리X
  
  -> 구조체를 사용해서 간단히 처리

- `구조체를 정의한다`: `int`나 `char` 같은 자료형을 하나 만듦
- structure(구조)의 약어이는 `struct`를 사용하여 정의

```
struct sawon {
  char name[10];
  char position[10];
  int pay;
}
```

### 236. Python 시퀀스 자료형

시퀀스 자료형(Sequence Type): 리스트, 튜플, range, 문자열 등 연속적으로 값이 이어진 자료형

- 리스트: 다양한 자료형 값을 연속적으로 저장, 필요에 따락 ㅐ수 늘리거나 줄이기 가능
- 튜플: 리스트처럼 요소를 연속적으로 저장하나, 요소의 추가, 삭제, 변경 불가능
- range: 연속된 숫자를 생성. 리스트, 반복문 등에서 많이 사용

### 237. 변수 개요/변수명 작성 규칙
변수: 컴퓨터가 명령 처리하는 도중 발생하는 값을 저장하기 위한 공간. 변할 수 있는 값을 의미함.
- 정수형, 실수형, 문자형, 포인터형


작성 규칙

1. 영문자, 숫자, 언더바 사용X

### 238. 가비지 콜렉터(Garbage Collector)

### 241. 비트 연산자
|연산자|의미|비고|
|---|---|---|
|&|and|모든 비트가 1일 때만 1|
|^|xor|모든 비트가 같으면 0, 하나라도 다르면 1|
|l|or|모든 비트 중 한 비트라도 1이면 1|

|>>|오른쪽 시프트|비트 오른쪽 이동|

▶ 정답: ①번 \
 ② 럼바우(Rumbaugh), ③ 부치, ④ Jacobson 방법임.

 ---
 - 객체 지향 분석(OOA): 시스템 요구사항을 객체, 클래스, 상속 등의 개념으로 모델링하는 방법론
 - 1990년대 초 Coad-Yourdon, Rumbaugh, Booch, Jacobson 등의 방법론이 등장해 UML의 기반이 되었다.
    - Coad-Yourdon: 객체 식별부터 시작해 E-R 다이어그램으로 객체 행위를 모델링. 주요 과정은 객체/클래스 찾기, 구조(상속/집합) 식별, 주제 정의, 속성 연결, 서비스(연산/메시지) 정의
    - Rumbaugh: OMT 접근 (객-동-기) 순서. 객체 모델링(객체 다이어그램으로 클래스/관계), 동적 모델링(상태 다이어그램으로 시간 흐름), 기능 모델링(DFD로 데이터 흐름)을 강조
    - Booch: 미시적(상세)과 거시적(전체) 개발 프로세스를 통합하며, 클래스/객체 식별 후 속성과 연산을 정의함. 요구사항에서 명사(객체)와 동사(동작)을 추출해 설계 중심으로 확장.
    - Jacobson: Use Case(사용 사례)를 강조해 사용자 관점에서 요구사항을 분석하며, 전체 소프트웨어 생명 주기를 다룸. 클래스 다이어그램 등으로 분석과 설계를 연결.

|방법론|주요 모델/도구|초점 영역|순서/특징|
|---|---|---|---|
|Coad-Yourdon|E-R 다이어그램|객체 구조/행위 모델링|5단계(SOSAS): 객체>구조>주제>속성>서비스|
|Rumbaugh|객체/상태/DFD 다이어그램|정적/동적/기능 모델|객-동-기 순|
|Booch|클래스/상태 다이어그램|클래스 속성/연산 정의|미시/거시 프로세스|
|Jacobson|Use Case 다이어그램|사용자 중심 요구 사항|생명주기 전체|
